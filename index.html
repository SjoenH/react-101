React 101

A Beginner's Guide to React

Based on the official React documentation at react.dev/learn

What is React?

  • A JavaScript library for building user interfaces
  • Developed by Facebook (now Meta)
  • Component-based architecture
  • Declarative programming paradigm
  • Virtual DOM for efficient updates

Why React?

  • Reusable Components: Build encapsulated components that manage their own state
  • Virtual DOM: Efficient updates and rendering
  • Large Ecosystem: Rich ecosystem of libraries and tools
  • Strong Community: Active community and excellent documentation
  • Job Market: High demand for React developers

Your First Component

Components are the building blocks of React applications.


function Welcome() {
  return <h1>Hello, World!</h1>;
}

// Or as an arrow function
const Welcome = () => {
  return <h1>Hello, World!</h1>;
};

Components must start with a capital letter!

JSX - JavaScript XML

JSX allows you to write HTML-like syntax in JavaScript


const element = <h1>Hello, World!</h1>;

// JSX with expressions
const name = 'React';
const greeting = <h1>Hello, {name}!</h1>;

// JSX attributes
const image = <img src="logo.png" alt="React Logo" />;

JSX Rules

  • Return a single root element (or use React.Fragment)
  • Close all tags (<img />, <br />)
  • Use camelCase for attributes (className, onClick)

// Good
function App() {
  return (
    <div>
      <h1 className="title">Welcome</h1>
      <img src="logo.png" alt="Logo" />
    </div>
  );
}

Displaying Data

Use curly braces {} to embed JavaScript expressions in JSX


function Profile() {
  const user = {
    name: 'Sarah Connor',
    imageUrl: 'https://example.com/sarah.jpg'
  };

  return (
    <div>
      <h1>{user.name}</h1>
      <img src={user.imageUrl} alt={user.name} />
      <p>Welcome back!</p>
    </div>
  );
}

Conditional Rendering

Show different content based on conditions


function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in.</h1>;
}

// Using ternary operator
function Status({ isOnline }) {
  return (
    <div>
      User is {isOnline ? 'online' : 'offline'}
    </div>
  );
}

Rendering Lists

Use map() to render arrays of data


function TodoList() {
  const todos = [
    { id: 1, text: 'Learn React' },
    { id: 2, text: 'Build an app' },
    { id: 3, text: 'Deploy to production' }
  ];

  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>{todo.text}</li>
      ))}
    </ul>
  );
}

Always provide a unique key prop!

Handling Events

React events are named using camelCase


function Button() {
  function handleClick(e) {
    e.preventDefault();
    alert('Button clicked!');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

// Inline event handler
function App() {
  return (
    <button onClick={() => alert('Hello!')}>
      Say Hello
    </button>
  );
}

State - Component Memory

State allows components to "remember" information and change over time


import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

useState Hook

The useState hook lets you add state to function components


import { useState } from 'react';

function Form() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');

  return (
    <form>
      <input 
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Your name"
      />
      <input 
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Your email"
      />
      <p>Hello, {name}! Your email is {email}</p>
    </form>
  );
}

State Updates

  • State updates are asynchronous
  • React may batch multiple state updates
  • Use functional updates for state that depends on previous state

function Counter() {
  const [count, setCount] = useState(0);

  // Functional update - recommended for dependent state
  const increment = () => setCount(prev => prev + 1);
  
  // Multiple updates
  const incrementTwice = () => {
    setCount(prev => prev + 1);
    setCount(prev => prev + 1);
  };

  return (
    <button onClick={incrementTwice}>
      Count: {count}
    </button>
  );
}

useEffect Hook

Handle side effects like API calls, subscriptions, and timers


import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    // Fetch user data when component mounts or userId changes
    fetch(`/api/users/${userId}`)
      .then(response => response.json())
      .then(userData => setUser(userData));
  }, [userId]); // Dependency array

  if (!user) return <div>Loading...</div>;

  return <div>Hello, {user.name}!</div>;
}

useEffect Cleanup

Clean up subscriptions and prevent memory leaks


import { useState, useEffect } from 'react';

function Timer() {
  const [time, setTime] = useState(new Date());

  useEffect(() => {
    const timer = setInterval(() => {
      setTime(new Date());
    }, 1000);

    // Cleanup function
    return () => {
      clearInterval(timer);
    };
  }, []); // Empty dependency array - runs once

  return <div>Current time: {time.toLocaleTimeString()}</div>;
}

Component Composition

Build complex UIs by composing simple components


function Avatar({ user }) {
  return <img src={user.avatar} alt={user.name} />;
}

function UserInfo({ user }) {
  return (
    <div>
      <Avatar user={user} />
      <h3>{user.name}</h3>
      <p>{user.email}</p>
    </div>
  );
}

function UserCard({ user, onEdit }) {
  return (
    <div className="card">
      <UserInfo user={user} />
      <button onClick={() => onEdit(user.id)}>
        Edit Profile
      </button>
    </div>
  );
}

Thinking in React

  1. Break the UI into components - Identify component boundaries
  2. Build a static version - No state, just props
  3. Identify state - What data changes over time?
  4. Identify where state should live - Common parent component
  5. Add inverse data flow - Pass callbacks down for state updates

Best Practices

  • Keep components small and focused
  • Use descriptive names for components and props
  • Extract reusable logic into custom hooks
  • Use TypeScript for better type safety
  • Follow the React DevTools for debugging
  • Test your components with Jest and React Testing Library
